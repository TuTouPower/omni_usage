下面是我建议的 **OmniUsage 主面板窗口完整设计方案**。核心结论是：

**Popup 是临时查看，内容决定窗口高度。**
**浮动窗口是常驻小面板，用户决定窗口高度。**

---

# OmniUsage 主面板窗口设计方案

## 1. 总体设计原则

OmniUsage 主面板只做一套内容 UI，不按平台做三套界面。

统一内容包括：

```text
顶部标题栏
总览 / 厂商 Tab
厂商卡片
概览 / 账号模式切换
多条用量统计
Token 统计图
底部状态栏
设置入口
```

区别只在外层窗口模式。

主面板外壳统一由 Main 进程里的 `main_panel_controller` 管理。托盘左键、设置切换、内容高度上报都进入这个控制器，再由控制器按配置决定使用 Popup 还是 Floating Window。

主面板支持两种外壳：

```text
弹出面板 Popup
浮动窗口 Floating Window
```

平台默认：

```text
macOS：弹出面板
Windows：浮动窗口
Linux：浮动窗口
```

用户可以在设置里手动切换。

主面板打开方式配置：

```text
mainPanelMode: system | popup | floating
```

解析规则：

```text
system → macOS 使用 Popup，Windows / Linux 使用 Floating Window
popup → 强制使用 Popup
floating → 强制使用 Floating Window
```

浮动窗口高度模式配置：

```text
floatingHeightMode: fixed | followContent
```

默认：

```text
mainPanelMode = system
floatingHeightMode = fixed
```

---

# 2. 两种窗口模式

## A. 弹出面板 Popup

适合 macOS 默认。

它的心智是：

> 点开看一眼，看完自动消失。

### Popup 行为

```text
点击菜单栏 / 托盘图标：打开主面板
再次点击图标：关闭主面板
点击外部：关闭
按 Esc：关闭
不显示系统标题栏
不显示关闭按钮
不允许用户手动调整高度
不需要小三角
```

### Popup 外观

```text
细长型白色圆角面板
轻阴影
无系统标题栏
无小三角
靠近菜单栏 / 托盘图标弹出
顶部仍然有 OmniUsage 标题、刷新、设置、更多按钮
```

### Popup 高度

Popup 使用现有的内容驱动高度机制。

```text
高度 = clamp(content_height, collapsed_min_height, workArea.height * 0.75)
```

也就是说：

```text
内容少 → 窗口变矮
展开卡片 → 窗口变高
内容超过 75% 屏幕可用高度 → 窗口不再变高，内部滚动
```

Popup 的高度永远自动适应内容，不提供固定高度选项。

---

## B. 浮动窗口 Floating Window

适合 Windows / Linux 默认。

它的心智是：

> 一个可以放在桌面角落的小监控窗口。

### 浮动窗口行为

```text
点击托盘图标：打开或聚焦主面板
再次点击托盘图标：隐藏主面板
点击外部：不关闭
窗口可拖动
固定高度模式下可手动调整宽度和高度
跟随内容模式下只允许手动调整宽度，不允许手动调整高度
记住窗口位置
记住用户调整后的宽度
固定高度模式下记住用户调整后的高度
关闭按钮 = 隐藏到托盘，不销毁窗口实例
不需要小三角
```

### 浮动窗口外观

```text
细长型白色圆角窗口
轻阴影
无小三角
顶部区域可拖动
可以有一个轻量关闭按钮
不做传统厚重标题栏
初始宽度使用默认值，例如 460px
用户可以手动调整宽度
高度由用户或设置模式决定
```

---

# 3. 浮动窗口高度设计

这是最关键的部分。

浮动窗口默认不应该自动适应内容，因为 Windows / Linux 用户更习惯稳定的小工具窗口。

所以浮动窗口高度有两个模式：

```text
保持窗口大小
跟随内容变化
```

默认：

```text
保持窗口大小
```

---

## A. 保持窗口大小，默认

这是 Windows / Linux 的默认行为。

### 行为

```text
窗口高度不随内容自动变高变矮
用户把窗口拖到多高，就保持多高
用户可以调整窗口宽度和高度
展开 / 折叠卡片只影响内部内容
内容超出窗口高度时，内部滚动
下次打开记住用户上次调整的宽度、高度和位置
```

### 重要规则

```text
不提供高度数字输入框
不让用户设置 720px / 800px
用户通过拖动窗口边缘调整高度
系统记住这个高度
不受内容自适应的 75% 高度限制
恢复窗口时必须保证窗口主体仍在当前屏幕可见区域内
```

也就是说，固定高度模式不是完全无约束恢复旧 bounds。

```text
用户手动 resize 时 → 尊重用户当前调整
下次打开 / 换屏 / 缩放变化后恢复时 → 如果旧位置或旧尺寸超出当前 workArea，就轻微 clamp 回可见区域
```

这样既保留“用户决定窗口高度”的心智，又避免外接屏拔掉、分辨率变小、任务栏位置变化、缩放比例变化后窗口跑到屏幕外或无法操作。

如果用户从来没有手动调整过高度，就使用默认浮动窗口高度，例如当前初始高度或一个合理默认值。

建议默认：

```text
宽度：460px
高度：720px 左右
```

但这个数字不要暴露给用户。

---

## B. 跟随内容变化，高级选项

这是给少数喜欢紧凑窗口的用户。

### 行为

```text
展开内容时窗口自动变高
折叠内容时窗口自动变矮
用户可以手动调整窗口宽度
用户不可以手动调整窗口高度
最大高度仍然限制为屏幕可用高度的 75%
超过后内部滚动
```

这个模式复用现有 Popup 高度机制：

```text
高度 = clamp(content_height, collapsed_min_height, workArea.height * 0.75)
```

但它不是浮动窗口默认行为。

---

# 4. 现有高度机制怎么使用

你现在的机制不需要废掉，只需要由 `main_panel_controller` 按窗口模式和高度模式分流。

统一数据流：

```text
Renderer mirror 测量内容高度
IPC 上报 content_height / collapsed_min_height
main_panel_controller 判断当前外壳和高度模式
按分支决定是否 setBounds
```

## Popup 模式

继续使用现有内容自适应高度机制。

```text
Renderer mirror 测量内容高度
IPC 上报 content_height / collapsed_min_height
Main clamp 到 75% 屏幕高度
BrowserWindow.setBounds 改真实窗口高度
```

Popup 的打开、关闭由“主面板打开方式”决定后的窗口模式承接；托盘左键本身只表达“打开主面板”，不直接绑定 Popup。

## 浮动窗口 + 跟随内容变化

也使用现有机制，但只由内容高度控制窗口高度，窗口宽度保留用户当前宽度。

```text
content_height → clamp → setBounds(height)
current_width → preserve
```

## 浮动窗口 + 保持窗口大小

不要根据 content_height 改 BrowserWindow 高度。

也就是说：

```text
Renderer 仍然可以测量内容高度
Main 仍然可以收到高度上报
但在“保持窗口大小”模式下，不响应内容高度改变去 setBounds
```

窗口尺寸和位置来自：

```text
用户当前手动调整后的窗口宽度、高度和位置
或上次保存的 floatingBounds
```

恢复保存的窗口 bounds 时需要做可见性修正：

```text
旧 bounds 仍在当前 workArea 内 → 原样恢复
旧 bounds 超出当前 workArea → clamp 到当前屏幕可见区域
旧屏幕不存在 → 移到当前主屏或托盘所在屏幕
```

内容超出时由内部滚动处理。

# 5. 设置页设计

位置：

```text
设置 > 常规 > 窗口
```

建议 UI：

```text
窗口

主面板打开方式
[ 跟随系统推荐 v ]

启动时最小化到托盘       开启
记住窗口位置             开启
窗口始终置顶             关闭

浮动窗口
窗口高度
[ 保持窗口大小 v ]
```

托盘左键行为不单独配置为 Popup 或浮动窗口。

```text
左键托盘图标永远表示：打开 / 聚焦主面板
主面板实际使用 Popup 还是浮动窗口，由“主面板打开方式”决定
右键托盘图标仍然打开托盘菜单
```

---

## 主面板打开方式选项

```text
跟随系统推荐
弹出面板
浮动窗口
```

说明文案：

```text
跟随系统推荐：macOS 使用弹出面板，Windows 和 Linux 使用浮动窗口。
```

保存主面板打开方式后立即生效。
如果主面板当前已经打开，Main 进程应立刻按新模式重建外壳。
切换时不复用旧外壳的 bounds，新窗口按新模式自己的定位和尺寸规则重新计算。

---

## 浮动窗口高度选项

只在当前模式是浮动窗口时显示。

```text
窗口高度
- 保持窗口大小
- 跟随内容变化
```

说明文案：

```text
保持窗口大小：展开内容时窗口不变高，内容在窗口内滚动。
跟随内容变化：窗口会根据展开和折叠自动调整高度。
```

不要出现：

```text
720px
最大高度 75%
content_height
collapsed_min_height
自适应高度算法
```

这些都是内部实现，不要暴露给用户。

# 6. 不同平台默认状态

## macOS 默认

实际模式：弹出面板
高度：跟随内容

可手动调整高度：否

## Windows 默认

```text

实际模式：浮动窗口
高度：保持窗口大小
点击外部关闭：否
可拖动：是
可手动调整高度：是
记住位置：是
小三角：不要
```

## Linux 默认

```text
打开方式：跟随系统推荐
实际模式：浮动窗口
高度：保持窗口大小
点击外部关闭：否
可拖动：是
可手动调整高度：是
记住位置：是
小三角：不要
```

# 7. 顶部栏设计

## Popup 顶部栏

```text
OmniUsage        刷新   设置   更多
```

不要放关闭按钮。
因为点击外部、Esc、再次点击托盘图标都可以关闭。

## 浮动窗口顶部栏

```text
OmniUsage        刷新   设置   更多   ×
```

这里的 `×` 不是退出程序，而是隐藏到托盘。

顶部栏同时作为拖动区域。
不要做厚重系统标题栏。

# 8. 交互规则

## 主面板打开方式

```text
左键托盘图标：打开 / 聚焦主面板
右键托盘图标：打开托盘菜单
```

左键不再表达“打开 Popup”或“打开浮动窗口”。
它只负责打开主面板。

主面板实际外壳由用户设置决定：

```text
跟随系统推荐 → macOS 使用 Popup，Windows / Linux 使用浮动窗口
弹出面板 → 使用 Popup
浮动窗口 → 使用 Floating Window
```

## Popup

```text
打开时靠近菜单栏 / 托盘图标
高度随内容变化
点击外部关闭
Esc 关闭
再次点击图标关闭
不记住用户调整高度
不允许手动 resize
```

## 浮动窗口

```text
第一次打开使用默认位置和默认高度
用户拖动后记住位置
用户调整宽度后记住宽度
固定高度模式下，用户调整高度后记住高度
跟随内容模式下，高度不由用户调整和持久化
再次打开恢复上次位置和尺寸
恢复时如果旧位置 / 旧尺寸不适合当前屏幕，则修正到可见区域
点击外部不关闭
点击关闭按钮隐藏到托盘，不销毁窗口实例
托盘图标再次点击显示 / 隐藏主面板
```

# 9. 实现注意事项

这份方案不是纯 UI 文案改动，落地时需要补齐以下边界：

```text
新增 main_panel_controller，统一管理主面板外壳生命周期
新增主面板打开方式配置 mainPanelMode
新增浮动窗口高度模式配置 floatingHeightMode
新增浮动窗口位置 / 尺寸持久化 floatingBounds
托盘左键逻辑改为打开主面板，再根据配置选择 Popup 或浮动窗口
设置保存 mainPanelMode 后，如果主面板已打开，立即切换外壳
切换外壳时按新模式重新计算 bounds，不继承旧外壳 bounds
Popup 点击外部关闭需要确认或补实现
窗口始终置顶配置需要在 Main 进程实际应用到主面板窗口
恢复浮动窗口 bounds 时必须按当前 workArea 做可见性修正
跟随内容模式下，浮动窗口只允许用户调整宽度，不允许调整高度
```

现有内容自适应高度机制仍然保留，但只服务于：

```text
Popup
浮动窗口 + 跟随内容变化
```

不会服务于默认的：

```text
浮动窗口 + 保持窗口大小
```

# 10. 验证要求

至少覆盖以下验证：

```text
模式解析：system 在不同平台解析为正确外壳
浮动窗口 bounds clamp：换小屏、旧屏幕不存在、任务栏 workArea 变化
高度上报分流：Popup 应用、Floating followContent 应用、Floating fixed 忽略
Windows / Linux 默认左键托盘打开 Floating Window
设置切换主面板打开方式后，已打开主面板立即切换外壳
Floating Window 点击关闭按钮只 hide，不销毁；再次左键托盘恢复
固定高度模式下展开卡片只触发内部滚动
跟随内容模式下高度随内容变化，且用户只能调整宽度
```

涉及 UI 的实现完成前，必须手工点击验证关键路径。

---

# 11. 不做的东西

不要做小三角。

不要让用户设置具体高度数字。

不要在设置里暴露：

```text
75%
mirror
ResizeObserver
IPC
BrowserWindow.setBounds
collapsed_min_height
content_height
```

不要默认让 Windows / Linux 浮动窗口跟随内容跳动。

不要给 Popup 提供固定高度。

不要把 Popup 和浮动窗口做成两套不同主界面。

---

# 12. 最终一句话方案

**OmniUsage 主面板内容保持统一；左键托盘永远打开 / 聚焦主面板；主面板外壳由设置决定。macOS 默认使用弹出面板，窗口高度跟随内容；Windows / Linux 默认使用浮动窗口，窗口高度保持用户手动调整后的大小，内容超出时内部滚动。浮动窗口可以切换为“跟随内容变化”，该模式下用户只能调整宽度，高度由内容自动决定。**
