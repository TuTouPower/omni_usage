# T3 Spec: 掩码长度 + 已存密钥可回看（产品已改）

## 产品决策（已拍板，推翻旧不变量）

**前端必须能看到已保存的密钥。**  
打开编辑表单时，从主进程 vault 解密后经 IPC 把明文交给渲染进程填入输入框；眼睛睁/闭对**真密钥**生效；隐藏时掩码长度 = 真实字符数。

旧规则「渲染进程永远只拿 hasSecret」在本功能上作废，实现时同步改：

- `docs/omni_powers/op_blueprint/specs/secret-vault.md` 最小暴露规则
- `domain.md` §9
- `prd.md` / `architecture.md` / `ipc.md` 中「secret 永不经 IPC 明文」表述

## 问题

1. 隐藏后星号数量应等于密钥长度（3 字符 → 3 掩码）。
2. 旧实现用固定占位，且无法回看 → 与新产品要求冲突。

## 期望行为

| 场景                       | 行为                                                                 |
| -------------------------- | -------------------------------------------------------------------- |
| 打开已配置 secret 的编辑页 | IPC 取明文 → input value = 真密钥；默认 `type=password`（闭眼）      |
| 闭眼                       | 浏览器 password 掩码，长度 = value.length                            |
| 睁眼                       | 可见完整明文                                                         |
| 未配置                     | 空输入框                                                             |
| 保存                       | 与现网一致：有改动则 `saveSecrets`；无改动可跳过写 vault（可选优化） |
| 关闭表单                   | 不强制清内存（Electron 进程内）；不写日志明文                        |

## 安全边界（诚实记录）

- **仍防**：仅拷贝 `secrets.vault` 无 `vault.key` 读不出。
- **新增暴露**：设置窗打开期间，明文在渲染进程内存；截图/开发者工具/恶意预加载可触及。用户已接受该代价。
- **日志**：scrubber 仍必须脱敏，明文禁止进 log。
- **范围**：仅设置/编辑表单按需拉取，**用量面板/托盘不拉密钥**。

## 验收

1. 保存 API Key `abc` → 关对话框 → 再打开编辑 → 闭眼 3 点，睁眼 `abc`。
2. CPA 管理密钥、SESSION_COOKIE 等同。
3. 新账号未保存：行为与现网一致。
4. `pnpm test` 覆盖 getSecrets IPC + 表单回填。

## 非目标

- 用量面板用量卡显示密钥。
- 导出文件加密（导出已是明文，见现 secret-vault）。
- 主口令二次确认（可后续增强，本 task 不做）。
