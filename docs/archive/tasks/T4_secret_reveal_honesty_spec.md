# T4 Spec: 密钥眼睛（在可回看前提下）

## 产品前提（已拍板）

保存后再打开，前端经 IPC 拿到 vault **明文**（见 T3）。因此「按眼睛看不到原密码」的根因被移除。

## 期望

1. 图标两态：`eye` 睁 = 显示明文，`eye_off` 闭 = `type=password`。
2. 打开编辑时 value 已是真密钥 → 睁眼必见原文。
3. 闭眼掩码长度 = 字符数（浏览器 password 或等长，与 T3 一致）。
4. 所有 secret 框具备眼睛（T2）。

## 验收

1. 已存密钥打开 → 闭眼等长掩码 → 睁眼完整明文。
2. 新输入同理。

## 非目标

- 不再做「无法回显」提示（旧 T4 方案废弃）。
- 不在用量面板展示密钥。

## 依赖

T3 IPC 回填完成前，本 task 无法单独验收「已存回看」。
