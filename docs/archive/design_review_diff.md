# OmniUsage 前端差异分析：设计 Demo vs 当前实现

> 生成日期：2026-06-02
> 对比基准：`docs/design/omni-usage/project/` (设计 Demo) vs `src/renderer/` (当前 Electron 实现)

## 1. 窗口尺寸与布局

| 项目     | 设计 Demo                                                 | 当前实现                                     | 差异                                                   |
| -------- | --------------------------------------------------------- | -------------------------------------------- | ------------------------------------------------------ |
| 窗口宽度 | 460px                                                     | 360px (BrowserWindow config)                 | **Demo 宽 100px**                                      |
| 窗口圆角 | 18px                                                      | 18px                                         | 一致                                                   |
| 窗口阴影 | 多层强阴影 (`0 20px 60px -12px ...`)                      | 同 token 定义                                | 一致                                                   |

## 13. CSS Token 对齐情况

当前 `globals.css` 的设计 token **与 Demo `omniusage.css` 高度一致**：

- ✅ 颜色变量名称和值完全匹配 (light + dark)
- ✅ 组件类名和结构基本匹配
- ✅ 进度条、骨架屏、菜单、对话框样式均匹配
- ⚠️ 混用 Tailwind CSS 变量 (`--border`, `--card`, `--muted-foreground` 等) 和自定义变量 (`--card-bg`, `--text-2` 等)
- ⚠️ SettingsForm / CpaConnectorSettings 等使用 Tailwind 风格变量，与主 UI token 不统一

## 14. 优先级建议

### 高优先 (影响核心体验)

1. **多账号平均用量** — 添加 L2 segmented switch 和平均用量视图
2. **卡片更新时间** — 在卡片头部显示相对更新时间
3. **窗口宽度** — 从 360px 调整到 460px

### 中优先 (功能完整性)

5. **拖拽排序** — 实现卡片拖拽 reorder
6. **右键菜单启停** — 实现「启用/关闭监控」实际功能
7. **强调色系统** — 持久化并实际应用 accent color 到 `--blue` 变量
8. **设置持久化** — 通知/刷新间隔/语言/置顶等设置写入配置
9. **品牌图标** — 替换占位 SVG 为真实 vendor logos

### 低优先 (视觉打磨)

10. **自定义托盘菜单 UI** — 用自定义 BrowserWindow 替代原生菜单
11. **设置项补全** — 清除缓存、导出 usage 数据、重置应用等功能实现
12. **CSS 变量统一** — 消除 Tailwind 变量与自定义变量混用
