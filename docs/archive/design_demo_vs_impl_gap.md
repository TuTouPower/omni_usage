# 设计 Demo vs 当前前端 —— 差异分析

> 对比对象：`docs/design/omni-usage/project/` (Demo) vs `src/renderer/` (实现)
> Token 面板已知关闭，不在本次分析范围内

---

## 1. 设置面板架构

| 项目     | Demo                                                              | 当前实现                                                      |
| -------- | ----------------------------------------------------------------- | ------------------------------------------------------------- |
| 打开方式 | `window.open()` 新窗口，独立 `OmniUsage Settings.html`            | 同路由切换 `#settings` → `SettingsView`，同一窗口             |
| 返回方式 | 左侧导航栏顶部 ← 按钮关闭 popup 回主面板                          | 内置返回按钮切回 `PopupView`                                  |
| 导航项   | 常规 / 账号 / 数据源(CPA才显示) / 外观 / 通知 / 数据与隐私 / 关于 | 常规 / 账号 / 外观 / 通知 / 数据与隐私 / 关于（无「数据源」） |

**差距**：

- Demo 的设置是独立窗口，不影响主面板交互。当前实现是路由切换，设置和主面板不能同时显示。
- 缺少 **数据源** 页面（CPA Manager 专属）。

## 2. 设置 — 账号页

| 项目              | Demo                                                                       | 当前实现                                        |
| ----------------- | -------------------------------------------------------------------------- | ----------------------------------------------- |
| 单账号厂商显示    | 整行：厂商 Logo + 备注名 + 脱敏 key + 开关 + 编辑 + 删除                   | 有 `SettingsForm`，结构不同                     |
| 多账号厂商        | 分组卡：厂商头 + 子行列表（每行有拖拽手柄 + 状态点 + 备注名 + key + 操作） | 有 `ProviderAccountList` / `ProviderAccountRow` |
| 来源标签          | CPA 账号显示「来自 CPA Manager」标签，直接添加显示「直接添加」             | 有 `source` 字段显示（`source.toUpperCase()`）  |
| CPA 模式隐藏/删除 | CPA 账号→隐藏按钮（eye_off）；直接添加→删除按钮（trash）                   | 无区分，统一删除                                |

**差距**：

- 缺少 **普通用户 / CPA 用户** 两种模式切换。
- CPA 账号的「隐藏」 vs 「删除」逻辑未实现。
- 账号行的字段布局（备注名 + 脱敏 key 并排）与 demo 有视觉差异。

## 3. 设置 — 数据源页（完全缺失）

Demo 有完整的数据源管理：

- **数据源列表**：CPA Manager 卡片，显示状态、URL、发现账号数、覆盖服务商、上次同步时间、同步/编辑/更多按钮
- **CPA Manager 详情页**（左右双栏布局）：
    - 左侧：连接配置（URL、API 密钥）、连接状态、同步设置（间隔、自动同步、失败通知）、同步范围（各服务商开关）、保存/移除按钮
    - 右侧：可折叠的已发现账号列表（按服务商分组）
- **添加数据源弹窗**：URL、密钥、同步范围选择、测试连接、保存并同步

当前实现完全没有这个页面。

## 5. 主面板 — 卡片交互

| 项目           | Demo                                 | 当前实现                                            |
| -------------- | ------------------------------------ | --------------------------------------------------- |
| 折叠状态       | 卡片折叠为小白条，只显示基础信息     | 有 `CollapsibleCard`，功能基本对齐                  |

**差距**：

- Demo 中禁用的卡片在主面板**不显示**（`if (disabledSet.has(key)) return`），当前实现仍显示灰色卡片。
- 删除功能未真正接入后端。

## 6. 主面板 — 用量条 (BarRow)

| 项目     | Demo                                             | 当前实现                                                 |
| -------- | ------------------------------------------------ | -------------------------------------------------------- |
| 标签     | 「5小时」「一周」                                | 使用 `window.name`，通常是「5 小时」「一周」或自定义名称 |
| 百分比   | 进度条右侧直接显示百分比                         | 有百分比显示                                             |
| 重置时间 | 进度条最右侧显示具体重置时间（如「今天 13:10」） | 显示「待重置」或「--」，无具体时间                       |
| 危险色   | `>= 85%` 自动变红                                | 有 `critical` / `warning` 状态                           |

**差距**：

- 重置时间列只显示「待重置」文字，demo 显示**具体时间**（如「今天 13:10」「5/18 21:00」）。这是个明显的视觉差距。

## 差异优先级排序

### P0 — 功能缺失

1. **设置面板应为独立窗口**：当前是路由切换，需要改为 `window.open()` 或 Electron `BrowserWindow`
2. **数据源页面缺失**：CPA Manager 的完整数据源管理（列表、详情、添加弹窗）
3. **CPA 模式区分**：账号页的「来自 CPA Manager」标签 + 隐藏 vs 删除逻辑

### P1 — 视觉差异

4. **重置时间列**：进度条右侧应显示具体重置时间，当前只有「待重置」
5. **禁用卡片行为**：demo 中禁用卡片在主面板隐藏，当前仍显示灰色状态
6. **Tab 右侧多余箭头**：移除 `tabs-chevron`

### P2 — 功能不完整

7. **删除 provider 未接入后端**：`delete_provider` 仅打日志
8. **添加账号弹窗**：demo 用网格+高级方式，当前是表单式

### P3 — 开发工具（可选）

9. **Tweaks 面板**：仅开发调试用，非生产需求
