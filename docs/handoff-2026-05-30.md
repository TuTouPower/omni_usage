## 四、未完成 / 待做

### 4.1 Tray 交互测试需要 rebuild（优先级：高）

**问题**：tray 测试的"点击关闭 popup"功能依赖新增的 IPC 通道 `test:tray-click`，但 `.vite/build/` 目录下的构建产物是旧的，不包含新代码。

**改动涉及文件**：

- `src/main/index.ts` — 新增 `ipcMain.handle("test:tray-click", ...)` + `E2E_WITH_TRAY` 环境变量支持
- `src/preload/index.ts` — 新增 `contextBridge.exposeInMainWorld("__test__", { trayClick: ... })`
- `src/shared/types/ipc.ts` — 新增 `TEST_TRAY_CLICK` 通道常量

**解决方法**：

```bash
# 方法1：用 electron-vite 重建（需先安装）
pnpm add -D electron-vite
pnpm exec electron-vite build

# 方法2：用 electron-forge 重建（会启动 app，Ctrl+C 退出即可）
electron-forge start
# 等看到窗口后 Ctrl+C

# 然后跑 E2E
pnpm test:e2e
```

**当前 `electron-vite build` 失败原因**：npx 缓存版本找不到本地 electron 模块。解决方案是安装到项目本地依赖（`pnpm add -D electron-vite`），而非用 npx。

### 4.2 Tray 测试还有一个技术问题（优先级：中）

`tray_interaction.spec.ts` 的第二个 test case "tray click closes open popup" 通过 IPC 调用 `tray.emit("click")` 来模拟点击。但 `page.evaluate()` 是在 **renderer 进程** 执行的，不是 main 进程。

目前的方案是：

1. main 进程注册 `ipcMain.handle("test:tray-click")` → 调用 `tray.emit("click")`
2. preload 暴露 `window.__test__.trayClick()` → 调用 `ipcRenderer.invoke("test:tray-click")`
3. 测试通过 `page.evaluate(() => window.__test__.trayClick())` 触发

这个方案在 rebuild 后应该能工作，但**未实际验证过**。如果 IPC 方案不行，退路是把 tray test 简化成只验证 "tray 模式下 popup 正常渲染"（不测点击关闭）。

## 六、ESLint 注意事项

项目有严格 ESLint 规则，写测试时常见坑：

| 规则                                | 解法                                                                     |
| ----------------------------------- | ------------------------------------------------------------------------ |
| `no-non-null-assertion`             | 用 `expect(x).not.toBeNull()` + optional chaining，或 helper `env_key()` |
| `require-await`                     | 回调不需要 await 时去掉 `async`                                          |
| `await-thenable`                    | `act()` 内必须 `await Promise.resolve()` 再调 resolve                    |
| `no-empty-function`                 | `() => {}` → `() => undefined`                                           |
| `react-hooks/rules-of-hooks`        | Playwright 的 `use` 参数被误判，需 file-level `/* eslint-disable */`     |
| `consistent-type-imports`           | `import()` type → `import type { X }`                                    |
| `non-nullable-type-assertion-style` | 不用 `as HTMLElement`，改 optional chaining                              |

## 七、pnpm test 只跑 vitest，不跑 E2E

**重要**：`pnpm test` = `vitest run`，只跑 unit + integration + smoke 测试。

E2E 测试需要单独命令：

```bash
pnpm test:e2e          # playwright, 需图形桌面
pnpm test:visual       # 视觉回归
pnpm test:packaged     # 打包 smoke, 需先 pnpm package
pnpm test:contract:live # 真 API, 无 key 全 skip
pnpm test:full         # pnpm test + pnpm test:contract:live
```
