## 总结：关键缺失项

### 高优先级（视觉/交互显著差异）

1. **面板宽度** — 设计 460px vs 实现 360px
2. **设置窗口模型** — 设计面板内切换 vs 实现独立窗口
3. **多账号二级分段** — `平均` / `N账号` 切换，展开账号明细
4. **Token 面板** — 完整的 token 趋势图面板被禁用
5. **卡片拖拽排序** — 完全缺失
6. **Tab 分隔线 + 渐隐** — 缺失
7. **账号状态点** — 卡片内账号无在线/离线指示

### 中优先级（细节差异）

8. **菜单定位** — 相对卡片 vs 鼠标位置
9. **托盘菜单** — 可视化窗口 vs 系统原生
10. **多账号展开动画** — `maDrawer` 动画缺失
11. **状态栏** — 设计无、实现有（需确认是否保留）
12. **每组添加账号按钮** — 设计有、实现缺
13. **关闭账号的 toggle 交互** — 实现有 bug
14. **减弱动画支持** — `prefers-reduced-motion` 缺失
15. **退出后 UI** — 设计有退出卡片、实现无

### 低优先级（文案/微调）

16. **状态行文案** — "重新登录" vs "重新配置" 等
17. **版本号** — 设计 1.4.2 vs 实现 1.0.0（正常）
18. **关于页 logo 尺寸** — 56 vs 64
19. **菜单毛玻璃饱和度** — 缺 `saturate(170%)`
20. **菜单项文案** — "启用/关闭" vs "关闭监控"
