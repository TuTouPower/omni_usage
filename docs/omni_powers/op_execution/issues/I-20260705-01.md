# I-20260705-01: 连接器脚本沙箱非真隔离

来源: opinit 归档提取（原 PLAN.md 已知限制）

严重度: P1下个spec前必修
标签: tech-debt, security

## 描述

第三方/用户连接器脚本（`connectors/` 下含 `user_dir`）运行在 Node `vm.runInContext` 沙箱（`src/main/core/connector/runtime.ts`）。Node `vm` **不是真正隔离环境**（官方文档明确）——沙箱仅冻结并注入 `ctx`，无法阻止刻意逃逸（如 `(0, eval)("this")` 触达主进程全局）。

**影响**：用户被诱导安装的恶意连接器脚本可逃逸到主进程 = 完全沦陷。

**现有缓解**：脚本禁止 `import`/`export`（`compile_script` 正则拦截）；运行超时 15s；HTTP/文件能力经 `ConnectorContext` 受控。

**待办（需评估，非小改）**：改用 `isolated-vm`（v8 真隔离）或连接器进**子进程**（进程级隔离 + IPC 能力收窄）。两者都需重构运行时与 `ConnectorContext` 边界。

**决策点**：是否接受额外依赖/子进程开销，换取对用户连接器的强隔离。

**blueprint 现状记录**：`op_blueprint/specs/connector-runtime.md` 沙箱段 + `architecture.md` §6。
