# AI设计工具提示词

_Started 2026-05-30 05:57 UTC_

---

## User

我希望其他 ai design 给我像你一样的一个个可交互的窗口，而不是平铺的界面，另外还要再 tweaks 里面允许用户打开各种报错极端状态，查看特殊情况下的用户提示和前端界面，还有跨平台工具不要显示单个平台独有的设计比如 mac win不同的系统托盘窗口最大化按钮，该怎么说 prompt

## Assistant

你是想要一段可以复用的「提示词」，让别的 AI 设计工具产出和我类似的成果。我帮你把三个核心诉求拆清楚，再给你一段可以直接复制的 prompt。

## 你的三个核心诉求

1. **可交互的「窗口」而非平铺界面** — 要把每个界面包在一个可拖动/可聚焦的窗口里，能并排对比、单独放大，而不是一张张静态平铺图。
2. **Tweaks 里要能切换异常/极端状态** — 报错、空状态、加载失败、断网、超长文本、权限被拒等，用来检查这些情况下的前端提示与界面。
3. **跨平台工具不要露出单一平台专属设计** — 不要画 Mac 的红黄绿交通灯、Win 的最小化/最大化/关闭按钮、系统托盘等任何能暴露具体操作系统的元素。

## 可直接复制的 Prompt

```
请把设计稿做成「可交互的窗口卡片」形式，而不是一张张平铺的静态界面：

1. 窗口化呈现
   - 每个界面放进一个独立窗口卡片里，支持拖动、聚焦、并排对比、点击放大查看。
   - 多个方案要能在同一画布上左右并排，方便我对比，而不是分散成多个文件或一长条平铺。

2. Tweaks / 控制面板要能切换状态
   - 加一个 Tweaks 面板，让我能手动切换界面的各种状态，至少包括：
     正常 / 加载中 / 空状态 / 请求失败报错 / 断网离线 / 权限被拒 /
     超长文本溢出 / 输入校验错误 / 接口超时 / 数据为空。
   - 每种异常状态都要展示对应的前端提示文案和界面表现，方便我检查极端情况下的用户体验。

3. 跨平台中立，不暴露具体操作系统
   - 这是跨平台工具，窗口外壳不要出现任何单一系统专属元素：
     不要 Mac 的红黄绿交通灯，不要 Windows 的最小化/最大化/关闭按钮，
     不要系统托盘、Dock、任务栏等。
   - 用一套中立、自定义的窗口控件，保证在 Mac / Windows / Linux 上观感一致。
```

## 几个让效果更好的小贴士

- **把「状态」当成第一类需求说出来**，很多工具默认只给「正常态」，你不点名要异常态它不会主动做。
- **明确说「并排对比、可放大」**，否则容易得到分散的多文件而不是一个可对比的画布。
- **「中立窗口外壳」这个词很关键** — 直接说「不要任何能暴露操作系统的元素」比逐个列举更保险，但你已经举的那两个例子（交通灯、最大化按钮）正好把意思钉死了，建议保留。

需要的话我也可以**直接帮你做一个这样的示例**（窗口化 + Tweaks 切换异常态 + 中立外壳），你看着实物再调整提示词会更准。要不要我做一版?
