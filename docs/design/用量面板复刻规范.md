# Agent Session Usage 面板 · 复刻规范（Spec）

> 本文档与参考实现 `index.html` 配套使用。
> **HTML 文件是视觉与交互的唯一事实来源（"做成这样"）；本文档定义架构意图与变更边界（"允许你怎么做"）。**
> 两者冲突时：视觉/交互以 HTML 为准，架构/工程实践以本文档为准。

---

## 1. 项目目标

构建一个面向开发者的本地使用统计面板，展示 **Claude Code / OpenCode** 两类 coding agent 的会话与 token 用量。单页应用，无后端，数据来自本地记录文件。

三个主数据（metric）：

| 主数据     | 含义                                             |
| ---------- | ------------------------------------------------ |
| Token 用量 | `input + output + cache_read + cache_write` 之和 |
| 会话数     | 去重后的 `session_id` 数量                       |
| 调用次数   | 记录条数（每条记录 = 一次 assistant 消息）       |

四个切面（dimension）：**时间、项目（directory）、模型（model）、会话（session）**。

## 2. 数据契约（Data Contract）

适配层输出的每条记录必须符合以下契约。**契约之后的所有聚合/图表代码不得感知原始数据来源。**

```ts
interface AgentSessionUsage {
    session_id: string; // 会话 UUID
    title: string | null; // 可空，空时显示 slug 或 session_id 前 8 位
    directory: string | null; // 工作目录绝对路径，空显示 "(unknown)"
    slug: string | null; // 会话 slug，可空
    version: string | null; // 工具版本号，如 "2.1.181"
    parent_session_id: string | null; // 非空 = sub-agent 会话
    message_id: string; // 消息 UUID
    role: string; // 固定 "assistant"
    timestamp: number; // **毫秒级** Unix 时间戳（注意：不是秒）
    model: string; // 模型名，如 "claude-sonnet-4.5"
    input_tokens: number; // 整数，缺失按 0
    output_tokens: number;
    cache_read_tokens: number;
    cache_write_tokens: number;
    // —— 以下为面板附加元数据，不属于原始 schema ——
    agent: "claude-code" | "opencode";
}
```

适配层硬性要求：

- `timestamp` 必须归一化为**毫秒**（源数据若是秒，×1000）；
- 数值字段缺失/非法时按 `0` 处理，不得产生 `NaN`；
- `title` / `slug` / `directory` / `version` 为 `null` 是合法状态，UI 有降级显示；
- 一个 `session_id` 的所有记录共享相同的会话级字段（title/directory/slug/version/agent/parent_session_id），以任一条为准；
- 会话内可能混用多个模型（约 12% 的记录模型不同），**不得**按"会话主模型"聚合。

参考数据源（适配层负责解析，此处仅提示）：

- Claude Code：`~/.claude/projects/**/*.jsonl`，每行一条消息，含 usage 字段；
- OpenCode：本地存储（SQLite/JSON），按 session 组织。

## 3. 架构拆分要求

不得交付单文件。建议结构（框架不限，React+TS / Vue / 原生 ES Module 均可）：

```
src/
  data/
    adapter.ts        # 读取源数据 → AgentSessionUsage[]（唯一允许感知数据来源的文件）
    mock.ts           # 可复现 mock 数据（固定种子），契约同 adapter
  lib/                # 纯函数，禁止依赖 DOM/框架
    filter.ts         # filtered(records, {agent, start, end})
    aggregate.ts      # bucketize(时间分桶)、topGroups(TopN+其他)、groupBy、hitRateOf
    format.ts         # fmtTok / fmtInt / fmtTime
  components/
    MetricDonut.ts    # 环图（4 张复用同一组件）
    BarChart.ts       # 主柱状图
    Heatmap.ts        # 时段热力图
    SessionTable.ts   # 会话明细表 + 分页
    RangePicker.ts    # 预设 + 日历弹层
    Segmented.ts      # 分段选择器（通用）
  theme/
    palette.ts        # 双主题调色板 + 模型/项目配色
    theme.css         # CSS 变量
```

`lib/` 内函数必须可独立单测。

## 4. 页面结构与行为

### 4.1 第一行：头部

- 左：标题「Agent Session Usage」；
- 右（依次）：工具筛选（全部工具 / Claude Code / OpenCode）、时间预设（24 小时 / 7 天 / 1 月）、主题切换（深色/浅色，localStorage 持久化）、📅 自定义按钮；
- 📅 点击弹出浮层：开始/结束两个 `datetime-local` + 应用按钮；应用后预设取消选中、按钮高亮；点击浮层外自动收起；选预设则取消自定义高亮。

### 4.2 第二行：四张环图（同一组件实例化四次）

| 卡片          | 中心数值                            | 色块维度               |
| ------------- | ----------------------------------- | ---------------------- |
| 总 Token 消耗 | 总 tokens                           | 模型（Top5 + 其他）    |
| 会话数        | 去重 session 数                     | 项目（全部，项目数少） |
| 调用次数      | 记录条数                            | 模型（Top5 + 其他）    |
| 缓存命中率    | `cache_read / (cache_read + input)` | 四类 token 构成        |

- **无图例**，hover 显示 名称 + 数值 + 百分比；"其他"块的 tooltip 需展开列出其中每个模型及数值；
- 每张卡片右上角环比小字：与**前一个等长时段**对比（▲/▼ 百分比，命中率为 pp 差值；前段无数据时显示"前段无数据"）；
- 环比分母与当前筛选联动（自定义 3 天 → 对比其前 3 天）。

### 4.3 第三行：柱状图 + 热力图

柱状图卡片头部：

- 左：主数据切换（Token / Session / 调用次数）；
- 右（依次）：时间粒度（小时/天，**仅横轴=时间时可见**）+ 横轴（时间 / 项目 / 会话）。

行为规则（交互不变量）：

| 主数据   | 纵轴               | 横轴可选                     | 色块维度         |
| -------- | ------------------ | ---------------------------- | ---------------- |
| Token    | tokens（fmtTok）   | 时间 / 项目 / 会话           | 模型 Top5 + 其他 |
| Session  | 去重会话数（整数） | **仅时间**（另两项禁用置灰） | 项目 Top5 + 其他 |
| 调用次数 | 整数               | 时间 / 项目 / 会话           | 模型 Top5 + 其他 |

- 横轴=时间：按粒度分桶；横轴=项目：按 metric 降序全列；横轴=会话：Top 20 会话；
- 无完整图例；tooltip 显示该柱总计 + 各色块数值 + "其他"内明细（最多 10 行）；
- 类目 > 14 时启用 dataZoom（inside + 底部滑块），滑块 `bottom: 8`、`height: 22`，标签不得被画布裁剪；
- 热力图：周一~周日 × 0~23h，亮度 = 当前主数据值，单色系渐变，无右上角说明文字；
- 24 小时范围下热力图只有 1~2 行有值，属预期行为，不做特殊处理。

### 4.4 第四行：会话明细表

- 按 `session_id` 聚合，列：会话（标题 + slug + sub-agent 标记）、工具/版本、工作目录、模型（会话内全部模型标签）、调用、Tokens（带内联条形）、缓存率、最近活跃；
- 点击表头排序（再点反向）；
- 分页：右下角依次为 每页条数下拉（10/20/50）、上一页、当前页码（`x / y`）、下一页；左下角不放任何信息；
- 筛选或排序变化时回到第 1 页。

## 5. 配色与主题

- 双主题（深色默认 / 浅色），通过 CSS 变量 + JS 调色板双轨实现，切换时图表重渲染；
- 模型配色全局一致：5 个常用模型高区分度亮色、7+ 个长尾模型低饱和色、**"其他"灰随主题变化**（深色 `#46506a` / 浅色 `#ccd3e0`）；
- 项目配色一套固定色板，全页面一致；
- 滚动条（含表格横向滚动条）颜色随主题：`scrollbar-color` + `::-webkit-scrollbar` 双写；
- 等宽字体 JetBrains Mono 用于数字、路径、标签。

参考色板见 HTML 中 `MODEL_COLORS` / `PROJECT_COLORS` / `PALETTES`。

## 6. 必须保持不变的交互清单（Invariants）

1. 所有图表/表格数据只经过 `filtered()` 一个出口，筛选全局联动；
2. Top5 + "其他"策略在环图、柱状图中一致，且"其他" tooltip 必须可展开明细；
3. Session 模式下横轴锁定"时间"，项目/会话按钮禁用；
4. 粒度选择器仅在横轴=时间时出现；切 24 小时预设自动切小时粒度，其余自动切天；
5. 环比始终对比"前一个等长时段"；
6. 主题切换立即重渲染所有图表，且持久化；
7. 数字格式：tokens 用 K/M/B 缩写，计数用千分位整数。

## 7. 验收 Checklist

- [ ] adapter 输出通过契约类型检查（含 null 字段、毫秒时间戳、缺省为 0）的单测；
- [ ] `lib/` 纯函数单测：分桶边界、Top5 分组、命中率、环比；
- [ ] 四张环图中心数值与明细表/柱状图总量一致（同一数据源交叉验证）；
- [ ] 三种主数据 × 三种横轴全部可切换，Session 模式的禁用逻辑正确；
- [ ] 自定义起止（精确到小时）生效且高亮日历按钮；
- [ ] 表格分页/排序/每页条数正确；
- [ ] 深/浅主题切换无残留色（重点：坐标轴、tooltip、热力图、dataZoom、滚动条、"其他"灰）；
- [ ] dataZoom 标签不被裁剪；
- [ ] 与参考实现 HTML 并排肉眼比对，布局/配色一致。

## 8. 变更边界

- **可自由实现**：组件拆分方式、框架选型、构建工具、适配层实现；
- **必须一致**：第 4、5、6 节的全部行为与视觉；与参考实现像素级比对通过验收；
- **禁止**：修改聚合语义（如按会话主模型聚合、把时间戳当秒处理、用记录数代替去重会话数）。
